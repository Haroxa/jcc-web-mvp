# AGENTS.md

本文件是 Codex 在本项目中的工作规则。开始修改前请先阅读本文。

## 项目概况

- 项目名称：JCC Web New
- 项目目标：为金铲铲水友赛主播提供可部署的直播辅助工具，覆盖主播端、管理端和游客公开页。
- 项目类型：可部署 Web 应用。
- 技术路线：TypeScript + React + Drizzle + SQLite，预留 Cloudflare Workers / D1 / R2 适配。
- 当前阶段：项目骨架已创建，正在进入核心功能开发前准备。
- UI 方向：朴素后台风格，优先功能完整、操作清楚、数据可靠。

## 当前关键业务范围

- 主播端：直播场次、定榜、推荐名单、锁牌、存票、截图、结算。
- 管理端：主播账号、代操作、资料管理、日志、异常检查。
- 游客端：公开存票榜和公开内容查看。
- 粉丝不登录、不注册。
- 管理员和主播登录。
- 管理员可以代主播操作。

## 常用命令

在项目根目录 `JCC-Web-New` 下执行：

```bash
npm install
npm run dev
npm run typecheck
npm run build
npm run db:generate
```

Cloudflare 相关命令：

```bash
npm run worker:dev
npm run worker:deploy
npm run db:migrate:local
npm run db:migrate:remote
```

注意：Cloudflare 命令依赖 `npx wrangler`，需要网络可用和 Cloudflare 配置完成。

## 本机网络和代理注意

当前 npm / pip 已按 Clash Verge 端口配置过：

- Clash 混合代理端口：`127.0.0.1:7897`
- npm proxy / https-proxy：`http://127.0.0.1:7897`
- pip proxy：`http://127.0.0.1:7897`

如果 Clash 退出，npm/pip 可能无法下载依赖。建议保持 Clash Verge 运行。

## 工作规则

- 修改前先确认影响范围，尤其是数据库 schema、迁移、票务计算和公开字段。
- 不要直接复用旧项目源码；旧项目只作为业务经验参考。
- 保持模块边界清晰，不把所有逻辑堆到一个文件。
- 票务、锁牌、名单生成等核心规则优先写成可测试的纯函数。
- 票务不直接硬改历史记录，优先用流水、修正、作废和日志。
- 截图默认不公开，公开由显式字段控制。
- 游客页只能展示公开字段，也只能按公开字段筛选。
- 数据库变更必须同步更新 `docs/06-数据库草案.md` 或迁移说明。
- 重要功能进展必须更新 `CHANGELOG.md` 和 `docs/项目状态.md`。
- 新问题或踩坑必须记录到 `docs/问题汇总与解决.md`。

## Git 提交规则

- 每完成一组明确任务后，应创建一次 Git 提交，方便后续追溯和回滚。
- 提交前先检查 `git status --short --branch`，确认没有误加入 `node_modules/`、`dist/`、密钥、真实数据或临时文件。
- 涉及代码时，优先运行 `npm run typecheck` 和 `npm run build`。
- 涉及文档时，至少确认相关文档时间、变更记录和状态说明已同步。
- 不把半成品调试过程拆成大量零碎提交；一个提交应该对应一个清晰目标。
- 提交信息使用中文短句，说明本次变更目标，例如 `初始化新项目骨架和项目文档`。
- 如果用户明确要求暂不提交，则只保留工作区改动并说明当前状态。

## 不要默认修改

- 不要删除用户数据、截图、迁移文件和配置文件。
- 不要提交真实账号、密码、token、Cloudflare id 或密钥。
- 不要默认改 npm/pip 全局代理配置，除非用户明确要求。
- 不要默认切换技术栈。
- 不要引入大型 UI 框架或复杂视觉方案，除非确实需要。
- 不要把旧项目的本地存储模型作为新项目实现方式。
