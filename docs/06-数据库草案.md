# 06-数据库草案

整理时间：2026-06-19 09:31

## 目标

本草案把数据模型进一步落到关系型数据库表结构。目标是兼容：

- 本地 SQLite。
- Cloudflare D1。
- 后续 Docker 服务器上的 PostgreSQL。

因此第一版表结构尽量使用通用字段类型和清晰关系，不使用过强的平台特性。

## 命名约定

- 表名使用复数蛇形命名，例如 `live_sessions`。
- 主键统一为 `id`。
- 时间字段统一使用 ISO 字符串或数据库时间戳，具体由实现阶段决定。
- JSON 字段在 SQLite/D1 中可用 `TEXT` 存 JSON；PostgreSQL 中可升级为 `jsonb`。
- 枚举先用字符串保存，后续再按技术栈决定是否做 enum 约束。

## 表结构草案

### accounts

账号表。只存管理员和主播账号，粉丝不登录。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `role` | `admin` / `streamer` |
| `streamer_id` | 主播账号关联主播 |
| `username` | 登录名，唯一 |
| `password_hash` | 密码哈希 |
| `display_name` | 显示名 |
| `status` | `active` / `disabled` |
| `active_session_id` | 当前有效登录会话 |
| `last_login_at` | 最近登录时间 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

关键索引：

- `unique(username)`
- `index(role)`
- `index(streamer_id)`

### account_sessions

账号会话表。用于支持登录态和单账号单设备在线。

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `account_id` | 关联账号 |
| `token_hash` | 登录 token 哈希，不保存明文 token |
| `status` | `active` / `revoked` |
| `user_agent` | 登录设备浏览器信息 |
| `ip_address` | 登录 IP |
| `created_at` | 创建时间 |
| `expires_at` | 过期时间 |
| `revoked_at` | 失效时间 |

关键索引：

- `index(account_id)`
- `index(token_hash)`
- `index(status)`

说明：

- 登录时会撤销同账号旧会话，并把 `accounts.active_session_id` 指向新会话。
- 校验登录态时同时检查 session 状态、过期时间、账号状态和 `active_session_id`。

- 作废流水不参与计算，但保留日志。

## 关键查询

### 查询公开存票榜

条件：

- 粉丝属于主播。
- `is_public_in_balance_board = true`。
- 公开模块 `balance_board` 开启。
- 只返回公开字段。

排序：

- `cached_ticket_balance desc`
- `display_name asc`

### 查询当前场次

条件：

- `streamer_id = 当前主播`
- `status in ('preparing', 'live', 'pending_settlement')`

排序：

- `started_at desc`

### 查询对局剩余可用五费

逻辑：

- 当前赛季可用五费。
- 排除当前对局中 `is_active_occupy = true` 的卡。
- 淘汰玩家的历史锁牌不参与排除。

### 计算粉丝余额

逻辑：

- 查询 `ticket_ledgers`。
- 过滤 `fan_id`。
- 过滤 `status = normal`。
- 只计算 `affects_balance = true`。
- 求和 `amount`。

## 数据完整性建议

- 所有关键业务修改写入 `audit_logs`。
- 票务不物理删除，只作废。
- 截图允许删除，但建议保留截图表记录，存储文件可删除。
- 粉丝拉黑不删除历史记录。
- 管理员代操作必须记录操作者。
