## P0：npm / pip 挂 VPN 后依赖下载失败

### 现象

- `npm install` 长时间卡住或超时。
- `npm view react version` 超时。
- `python -m pip index versions pip` 报代理连接失败或 TLS 中断。

### 原因

Clash Verge 当前混合代理端口是：

```text
127.0.0.1:7897
```

但旧配置中存在多个不一致端口：

```text
npm / 环境变量：127.0.0.1:7890
pip：127.0.0.1:10809
```

命令行工具会连接错误端口，导致依赖下载失败。

### 已执行解决

npm：

```text
proxy = http://127.0.0.1:7897
https-proxy = http://127.0.0.1:7897
registry = https://registry.npmmirror.com/
```

pip：

```text
global.proxy = http://127.0.0.1:7897
global.index-url = https://pypi.org/simple
```

### 验证结果

- `npm view react version` 成功。
- `python -m pip index versions pip` 成功。

### 注意事项

- Clash Verge 需要保持运行。
- 如果 Clash 混合端口变化，需要同步更新 npm / pip 代理。
- 如果退出 Clash，固定代理端口会失效。

## P1：pip 配置文件 BOM 导致无法读取

### 现象

执行 `python -m pip config list` 报：

```text
Configuration file could not be loaded.
File contains no section headers.
```

### 原因

PowerShell 写入 `pip.ini` 时使用了带 BOM 的 UTF-8，pip 读取失败。

### 解决

使用无 BOM UTF-8 重写 pip 配置文件。

### 注意事项

后续如果手动编辑 `pip.ini`，建议用无 BOM UTF-8。

## P1：依赖安装后生成构建产物

### 现象

项目目录出现：

- `node_modules/`
- `dist/`
- `*.tsbuildinfo`
- Vite/Drizzle 生成的 `.js` 文件。

### 当前判断

这些属于开发产物或编译产物，不应作为主要源码维护对象。

### 处理建议

- `node_modules/`、`dist/`、`*.tsbuildinfo` 应被 `.gitignore` 忽略。
- TypeScript 源文件为主，生成的 `.js` 文件不应提交。

### 已执行解决

- `tsconfig.node.json` 增加 `noEmit: true`。
- `.gitignore` 增加：
  - `drizzle/*.js`
  - `worker/*.js`
  - `vite.config.js`
  - `drizzle.config.js`
- 从 Git 跟踪中移除已生成的 `drizzle/schema.js` 和 `worker/index.js`。

## P1：Cloudflare Pages 更新后显示空白

### 现象

旧项目线上地址更新后显示空白。

### 原因

旧项目是纯静态页面，Cloudflare Pages 之前可以直接发布根目录。新项目是 Vite + React 应用，必须先执行构建。若仍发布根目录，线上 `index.html` 会引用 `/src/main.tsx`，浏览器无法直接运行该源码，导致页面空白。

### 解决

Cloudflare Pages 构建设置应调整为：

```text
Build command: npm run build
Build output directory: dist
Root directory: /
```

仓库侧已补充：

```text
wrangler.toml: pages_build_output_dir = "dist"
package.json: pages:deploy
README.md: Cloudflare Pages 设置说明
```

### 追加修正

`wrangler.toml` 原本同时包含 Pages 输出目录、Worker 入口、D1/R2 占位绑定。为避免 Pages 构建解析配置时受到 Worker 配置影响，已拆分为：

```text
wrangler.toml：仅用于 Pages，保留 pages_build_output_dir = "dist"
wrangler.worker.toml：预留 Worker API、D1、R2 配置
```

如果 Cloudflare Pages 再次部署成功，线上 `index.html` 应引用：

```text
/assets/index-*.js
/assets/index-*.css
```

而不应再引用：

```text
/src/main.tsx
```

### 验证

本地执行：

```powershell
npm run build
```

已通过。

### 当前阻塞

尝试使用 Wrangler 手动部署时失败：

```text
In a non-interactive environment, it's necessary to set a CLOUDFLARE_API_TOKEN environment variable
```

需要二选一：

- 在 Cloudflare Pages 后台修改构建设置后让 Git 自动部署。
- 配置 `CLOUDFLARE_API_TOKEN` 后使用 `npm run pages:deploy` 手动部署。
