# 项目状态

最后更新：2026-06-16 15:31

## 项目类型和技术栈

- 类型：纯静态网页 MVP
- 技术栈：HTML、CSS、原生 JavaScript、浏览器本地存储
- 启动方式：直接打开 `index.html`
- 线上地址：https://jcc-web-mvp.pages.dev/
- 自动化测试：暂无
- 构建流程：暂无

## 当前目标

为金铲铲水友赛提供一个轻量记录工具，优先验证截图管理、锁牌管理、存票管理是否适合真实自定义比赛场景。

功能路线和优先级见 `docs/功能设计安排.md`。

## 功能状态

| 模块 | 状态 | 当前说明 | 下一步 |
| --- | --- | --- | --- |
| 截图管理 | 已实现 | 可上传截图、填写玩家/标签/备注、搜索和删除 | 用真实水友赛数据测试浏览器存储上限 |
| 锁牌管理 | 已实现 | 支持锁牌、释放、重置、彩色分类标签、按分类筛选添加、淘汰保留历史锁牌、自动排名、玩家和五费显示模式切换；本场玩家固定 8 槽位，缺少玩家会占位；五费支持按赛季快速配置；支持当前对局名称、开始新对局、保存和恢复对局快照；淘汰排名计算和展示排序已修正；添加五费展开布局已修复 | 用真实淘汰顺序和本场配置流程测试是否顺手 |
| 存票管理 | 已实现 | 可记录存票/取票、备注，并按余额排序；只读取本场参与玩家 | 测试大量记录下的操作体验 |
| 基础资料 | 已实现 | 可维护候选玩家、昵称/抖音名/微信名/游戏名、五费外号、五费分类、赛季、标签、备注；列表支持顺序编号、筛选、页码分页和固定高度 | 验证旧五费标签迁移为赛季字段后是否符合预期 |
| 数据导入导出 | 已实现 | 支持 JSON 导出和导入 | 验证异常 JSON 的提示是否足够清楚 |
| 历史对局 | 初版实现 | 可保存、恢复、删除当前对局快照；当前只包含本场玩家、本场五费、当前对局名称和锁牌状态 | 后续评估是否纳入截图和存票流水 |
| 免费部署 | 已完成 | 已部署到 Cloudflare Pages | 收集不同网络下的访问反馈 |
| 多人协作 | 暂缓 | 当前不包含登录、房间码和云同步 | 只有真实使用证明需要时再设计 |

## 未验证项

| 项目 | 未验证原因 | 建议验证方式 |
| --- | --- | --- |
| 手机端长时间录入体验 | 还未经过真实比赛使用 | 用手机横屏完成一场模拟记录 |
| 大量截图存储 | 浏览器本地存储容量有限 | 连续上传多张截图，观察保存和导出表现 |
| 五费批量解析 | 不同用户复制文本格式可能不同 | 用真实赛季名单和水友自定义名单测试解析结果 |
| 本场启用设置 | 玩家和可选五费每局可能不同；已拆分赛季筛选和标签筛选 | 用一场 8 人局测试缺位、添加、移除、按赛季快速配置五费后锁牌和存票是否符合预期 |
| 五费赛季字段 | 刚完成旧数据兼容迁移 | 检查旧 `tags` 中第一个标签是否正确进入赛季字段，剩余标签是否保留 |
| 基础资料分页 | 刚调整为页码分页和固定高度，还未经过大量数据验证 | 导入 30 个以上玩家和五费，测试搜索、翻页、保存是否稳定 |
| 淘汰排名 | 真实比赛可能存在误操作恢复存活 | 模拟多人淘汰、恢复存活、再次淘汰，检查名次和锁牌可用池 |
| 历史对局恢复 | 刚加入快照恢复，还未经过真实使用 | 保存一局、修改当前配置、恢复历史局，检查本场玩家、五费和锁牌是否一致 |
| 当前对局流程 | 刚加入开始新对局入口 | 设置对局名称、开始新对局、保存历史，检查名称和锁牌重置是否符合预期 |
| 多名称展示 | 玩家可能缺少游戏名、抖音名或微信名 | 切换不同展示模式，检查是否能正确回退到昵称 |
| 国内访问稳定性 | 已部署但还没有多人反馈 | 让不同网络用户访问 `https://jcc-web-mvp.pages.dev/` |
| 数据恢复流程 | 目前只确认有导入导出功能 | 导出后清空数据，再导入检查恢复结果 |

## 当前维护原则

- 先保持本地可用和静态可部署。
- 不急着做登录、后端、数据库和多人实时协作。
- 每次重要变更更新 `CHANGELOG.md`。
- 涉及项目规则或状态变化时，同步更新本文档。
