# 项目状态

最后更新：2026-06-10 12:39

## 项目类型和技术栈

- 类型：纯静态网页 MVP
- 技术栈：HTML、CSS、原生 JavaScript、浏览器本地存储
- 启动方式：直接打开 `index.html`
- 线上地址：https://jcc-web-mvp.pages.dev/
- 自动化测试：暂无
- 构建流程：暂无

## 当前目标

为金铲铲水友赛提供一个轻量记录工具，优先验证截图管理、锁牌管理、存票管理是否适合真实自定义比赛场景。

## 功能状态

| 模块 | 状态 | 当前说明 | 下一步 |
| --- | --- | --- | --- |
| 截图管理 | 已实现 | 可上传截图、填写玩家/标签/备注、搜索和删除 | 用真实水友赛数据测试浏览器存储上限 |
| 锁牌管理 | 已实现 | 支持锁牌、释放、重置；只读取基础资料中本场参与玩家和本场可用五费 | 下一轮补排名和淘汰自动排名 |
| 存票管理 | 已实现 | 可记录存票/取票、备注，并按余额排序；只读取本场参与玩家 | 测试大量记录下的操作体验 |
| 基础资料 | 已实现 | 可维护候选玩家、本场参与、五费分类、本场可用、标签、备注，支持批量解析文本 | 验证真实赛季五费文本复制后的解析效果 |
| 数据导入导出 | 已实现 | 支持 JSON 导出和导入 | 验证异常 JSON 的提示是否足够清楚 |
| 免费部署 | 已完成 | 已部署到 Cloudflare Pages | 收集不同网络下的访问反馈 |
| 多人协作 | 暂缓 | 当前不包含登录、房间码和云同步 | 只有真实使用证明需要时再设计 |

## 未验证项

| 项目 | 未验证原因 | 建议验证方式 |
| --- | --- | --- |
| 手机端长时间录入体验 | 还未经过真实比赛使用 | 用手机横屏完成一场模拟记录 |
| 大量截图存储 | 浏览器本地存储容量有限 | 连续上传多张截图，观察保存和导出表现 |
| 五费批量解析 | 不同用户复制文本格式可能不同 | 用真实赛季名单和水友自定义名单测试解析结果 |
| 本场启用设置 | 玩家和可选五费每局可能不同 | 用一场 8 人局测试启用/禁用后锁牌和存票是否符合预期 |
| 国内访问稳定性 | 已部署但还没有多人反馈 | 让不同网络用户访问 `https://jcc-web-mvp.pages.dev/` |
| 数据恢复流程 | 目前只确认有导入导出功能 | 导出后清空数据，再导入检查恢复结果 |

## 当前维护原则

- 先保持本地可用和静态可部署。
- 不急着做登录、后端、数据库和多人实时协作。
- 每次重要变更更新 `CHANGELOG.md`。
- 涉及项目规则或状态变化时，同步更新本文档。
