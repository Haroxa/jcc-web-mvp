# 02-已有功能与页面清单

整理时间：2026-06-16

## 已实现内容

### 截图管理

已实现：

- 上传图片。
- 记录玩家、标签和备注。
- 按文本搜索截图记录。
- 删除单条截图记录。
- 图片以浏览器本地数据方式保存。

当前价值：

- 能快速记录精彩对局和备注。
- 适合作为“截图管理”最小验证版本。

限制：

- 图片保存在浏览器本地，容量和稳定性受浏览器影响。
- 没有和具体历史对局强绑定。
- 没有云端同步。

### 锁牌管理

已实现：

- 当前对局名称和开始时间。
- 当前赛季记录。
- 本场配置默认展开。
- 本场玩家配置。
- 本场五费卡配置。
- 按赛季快速配置五费。
- 玩家最多按 8 个本场席位展示。
- 缺少玩家时显示占位。
- 五费卡按正常五费、解锁五费、可选五费分类。
- 分类颜色区分。
- 支持玩家显示昵称、游戏名、抖音名、微信名。
- 支持五费显示正式名或外号。
- 已锁五费和可用五费摘要。
- 玩家状态：存活、淘汰。
- 淘汰玩家排名自动计算。
- 淘汰玩家锁牌记录保留。
- 淘汰玩家的锁牌记录可修改，且不占用存活玩家可选池。
- 存活玩家之间的五费选择互斥。
- 当前对局快照保存、恢复、删除。

当前价值：

- 已经覆盖水友赛追五费卡的核心场景。
- 本场配置从基础资料中独立出来，这个方向值得保留。

限制：

- 锁牌页面承担了过多功能。
- 表格布局在移动端仍有压力。
- 淘汰排名逻辑之前多次修复，说明这块规则容易出错。
- 历史快照仍是初级版本。

### 存票管理

已实现：

- 选择玩家。
- 记录存票或取票。
- 填写票数和备注。
- 计算每个玩家当前总票数。
- 按总票数排序。
- 展示存取票历史。
- 可重置存票数据。

当前价值：

- 可以验证“存票/取票流水 + 当前余额排行”这个基础模型。

限制：

- 尚未支持一场直播多局游戏。
- 尚未支持下午场、晚上场。
- 尚未支持定榜规则。
- 尚未支持榜五加两个新粉、无新粉顺延榜六榜七等流程。
- 尚未支持一场直播结束后的统一结算。

### 基础资料

已实现：

- 玩家资料维护。
- 玩家字段：昵称、抖音名、微信名、游戏名、备注。
- 五费卡资料维护。
- 五费字段：名称、外号、分类、赛季、标签、备注。
- 玩家批量文本解析。
- 五费批量文本解析。
- 搜索筛选。
- 分页。
- 固定高度表格区域。
- 五费分类颜色。
- 重置默认资料。

当前价值：

- 已经把“资料维护”和“本场选择”拆开，这个设计值得参考。
- 赛季字段从标签中拆出来，是一次重要的结构升级。

限制：

- 基础资料仍直接影响当前本场配置，边界还不够清晰。
- 文本解析格式仍依赖手工输入习惯。
- 没有导入校验报告。

### 数据导入导出

已实现：

- 导出当前本地数据为 JSON。
- 从 JSON 导入数据。
- 清空本地数据。

当前价值：

- 在没有后端的情况下提供了最低限度的备份能力。

限制：

- 需要用户手动操作。
- 无法跨设备自动同步。
- 没有版本化数据迁移提示。

## 半成品内容

- 历史对局：能保存和恢复快照，但缺少详情页、统计摘要、导出和与截图/存票的关联。
- 当前对局：有名称、开始时间、赛季，但缺少完整的开始、进行中、结束、归档生命周期。
- 存票管理：有基础流水和余额排行，但还没有贴合直播定榜和结算规则。
- 移动端适配：已有基础响应式样式，但锁牌表格和配置区仍需要继续打磨。
- 数据模型迁移：已有兼容旧字段的逻辑，但还没有独立的数据版本管理。
