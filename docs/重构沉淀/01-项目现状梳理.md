# 01-项目现状梳理

整理时间：2026-06-16

## 项目当前是什么

旧项目是一个面向金铲铲水友赛的小型网页工具，项目名为“水友赛记录助手 MVP”。它的目标是把直播水友赛中容易混乱的记录工作先用一个轻量网页跑通，包括截图记录、锁牌记录、存票记录、玩家资料和五费卡资料维护。

项目目前是一个纯前端静态网页，可以直接打开 `index.html` 使用，也已经部署到 Cloudflare Pages：

- 线上地址：https://jcc-web-mvp.pages.dev/

## 技术栈

- HTML：`index.html` 负责页面结构和功能入口。
- CSS：`styles.css` 负责页面布局、表格、标签、折叠面板和移动端基础适配。
- 原生 JavaScript：`app.js` 负责所有状态、渲染、事件、业务规则和数据读写。
- 浏览器本地存储：使用 `localStorage` 保存数据。
- 无框架、无构建工具、无后端、无数据库、无登录。
- 版本管理：Git。
- 免费部署：Cloudflare Pages。

## 启动方式

当前项目没有构建步骤。

本地使用方式：

1. 打开项目目录。
2. 双击或用浏览器打开 `index.html`。
3. 数据保存在当前浏览器本地。

线上使用方式：

1. 访问 Cloudflare Pages 地址。
2. 同一浏览器可继续读取本地保存的数据。
3. 更换浏览器、设备或清理浏览器数据后，本地数据不会自动同步。

## 主要页面

项目是单页应用，通过顶部标签切换功能区：

- 截图管理
- 锁牌管理
- 存票管理
- 基础资料

其中锁牌管理是当前迭代最多、业务规则最复杂的页面。

## 当前状态判断

这个旧项目已经不是纯想法，而是一个能使用、能部署、能验证核心流程的 MVP。它适合用来观察真实操作习惯、沉淀业务规则和发现页面结构问题。

但它还不是适合长期扩展的正式系统。主要原因是业务规则、数据结构、页面渲染和事件处理都集中在一个较大的 `app.js` 文件中，后续继续堆功能会越来越难维护。
