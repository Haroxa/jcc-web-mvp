# 03-页面流程草图

整理时间：2026-06-18

## 总体导航

第一版建议分为三个入口：

- 主播端：直播时高频操作。
- 管理端：账号、资料、辅助管理、日志。
- 游客端：公开存票榜和公开内容查看。

## 主播端页面

### 今日工作台

目标：主播一进来就能继续当前直播。

核心内容：

- 当前主播和当前场次。
- 今日场次状态：未开始、进行中、待结算、已结算。
- 快捷入口：定榜、锁牌、存票、截图、结算。
- 异常提醒：未结算、负数余额、待确认名单、未完成对局。

主要流程：

```text
打开工作台
  -> 没有进行中场次：创建直播场次
  -> 有进行中场次：进入当前场次
  -> 根据直播阶段进入定榜/锁牌/存票
```

### 直播场次页

目标：管理一场下午场/晚上场。

核心内容：

- 场次基础信息。
- 规则模板：定榜七候选 / 定榜五顺延。
- 模式：规则水友赛 / 无规则模式。
- 定榜列表。
- 对局列表。
- 票务预结算。
- 结束并结算。

关键操作：

- 创建定榜。
- 创建对局。
- 切换无规则模式。
- 进入结算确认。

### 定榜页

目标：记录每次定榜，并生成推荐名单。

核心内容：

- 定榜方式：定榜七候选 / 定榜五顺延 / 手动。
- 定榜条目：名次、粉丝、抖音名、礼物钻、取票、调整、总竞争票、状态。
- 新粉数量。
- 上车/待定/等下一把状态。
- 备注和截图。

主要流程：

```text
新增定榜
  -> 输入榜单条目和票数
  -> 录入本次取票/调整
  -> 系统计算总竞争票
  -> 生成推荐名单
  -> 主播手动调整
  -> 确认名单
  -> 创建对局
```

设计重点：

- 总竞争票可手动调整。
- 定榜期间可以随时改票。
- 生成名单不是最终裁决，主播确认后才进入对局。

### 对局锁牌页

目标：一局游戏中快速记录锁牌变化。

核心内容：

- 8 个席位。
- 席位身份：主播、老粉、新粉、补位、手动。
- 本局游戏名/备注。
- 存活/淘汰状态。
- 已锁五费。
- 剩余可用五费。
- 淘汰排名。

主要流程：

```text
确认名单创建对局
  -> 配置本局五费卡池
  -> 玩家锁牌
  -> 存活玩家锁牌互斥
  -> 玩家淘汰
  -> 淘汰玩家记录保留但释放占用
  -> 保存对局
  -> 新粉可转老粉
```

无规则模式：

- 不显示锁牌主流程。
- 可以记录参与人、游戏名、备注和截图。

### 存票页

目标：管理粉丝长期余额和本场票务变化。

核心内容：

- 粉丝搜索。
- 当前余额。
- 本场预览余额。
- 存票、取票、现刷、修正、回退。
- 本场流水。
- 作废入口。
- 结算确认。

主要流程：

```text
选择粉丝
  -> 记录存票/取票/现刷
  -> 关联场次或定榜
  -> 直播中显示预览余额
  -> 直播结束结算确认
  -> 入账或回退
```

设计重点：

- 修改通过新增修正记录。
- 作废保留日志。
- 取票未上车可标记未使用并结算回退。

### 截图页

目标：保存定榜、精彩、违规或其他截图。

核心内容：

- 上传截图。
- 类型：定榜、精彩、违规、其他。
- 关联：场次、对局、粉丝。
- 是否公开。
- 备注。
- 删除。

规则：

- 默认不公开。
- 公开后游客可见。
- 违规截图也可以公开，取决于公开状态。
- 单张推荐 5MB 限制。

### 基础资料页

目标：维护长期资料。

内容：

- 粉丝资料。
- 粉丝状态。
- 五费卡。
- 赛季。
- 规则模板。

## 管理端页面

### 管理首页

核心内容：

- 主播列表。
- 今日进行中场次。
- 待结算场次。
- 最近异常。
- 快捷进入主播详情。

### 主播详情页

核心内容：

- 主播资料。
- 当前场次。
- 历史场次。
- 粉丝资料。
- 存票汇总。
- 截图。
- 日志。

管理员可以代主播操作。

### 账号管理页

核心内容：

- 创建主播账号。
- 停用账号。
- 重置登录信息。
- 查看最近登录。
- 单设备在线状态。

### 操作日志页

核心内容：

- 按主播、操作者、动作、时间筛选。
- 查看前后变化。
- 查看备注。

## 游客端页面

### 公开首页

目标：未登录游客查看公开内容。

内容：

- 主播名称。
- 公开模块入口。
- 公开存票榜。
- 公开截图或说明。

### 公开存票榜

内容：

- 公开粉丝名。
- 存票余额。
- 排名。
- 分页。
- 筛选。

规则：

- 只能展示公开字段。
- 只能按公开字段筛选。

### 公开内容页

内容：

- 公开截图。
- 公开说明。
- 后续可扩展公开对局历史。

## 页面优先级

第一优先级：

- 主播工作台。
- 直播场次页。
- 定榜页。
- 存票页。
- 对局锁牌页。
- 管理端主播详情。
- 游客公开存票榜。

第二优先级：

- 截图页。
- 操作日志页。
- 账号管理页。
- 基础资料页。

后续增强：

- 公开截图墙。
- 数据导出。
- 异常看板。
- OBS/直播展示页。
