# 风险清单

最后更新：2026-06-18 18:18

## 风险等级

- `P0`：可能造成重要数据丢失、公开泄露、核心流程不可用。
- `P1`：影响核心功能正确性或真实直播使用。
- `P2`：影响体验、维护性或局部功能。
- `P3`：低风险优化项。

## 当前风险

| 等级 | 风险 | 影响 | 当前状态 | 建议 |
| --- | --- | --- | --- | --- |
| P0 | 票务流水计算错误 | 存票余额错误，影响信任 | 未实现 | 票务规则必须单测覆盖 |
| P0 | 结算误操作 | 长期余额被错误入账或回退 | 未实现 | 结算前预览，结算需确认，写日志 |
| P0 | 公开字段泄露 | 抖音名、备注、违规记录被游客看到 | 未实现 | 游客页只读公开字段，筛选字段也受控 |
| P0 | 管理员代操作无日志 | 事后无法追责 | 未实现 | 管理员代操作必须写日志 |
| P1 | 锁牌释放占用错误 | 剩余五费显示错误 | 未实现 | 锁牌规则写纯函数和测试 |
| P1 | 拉黑状态处理不完整 | 拉黑人仍可上车 | 未实现 | 推荐名单生成前检查状态 |
| P1 | 图片存储占用过大 | 免费额度或服务器磁盘被占满 | 未实现 | 单张 5MB 限制，允许删除 |
| P1 | Clash 端口变化 | npm/pip 依赖下载失败 | 已遇到 | 端口变化时同步 npm/pip 配置 |
| P1 | Cloudflare 平台限制 | Worker/D1/R2 免费额度不够 | 待验证 | 保留 Docker 云服务器备选 |
| P2 | 手机端操作效率不足 | 直播中操作慢 | 待设计 | 朴素后台基础上保证响应式 |
| P2 | 规则例外过多 | 推荐名单不能覆盖现场情况 | 已知业务特点 | 推荐结果必须支持手动调整和备注 |
| P2 | 生成文件混入源码 | 维护混乱 | 已出现可能 | 检查 `.gitignore` 和保留策略 |
| P2 | 未使用 Git 管理版本 | 后续难以回滚和追踪 | 已初始化 Git | 首次提交前检查范围 |

## 发布前检查

- `npm run typecheck` 通过。
- `npm run build` 通过。
- 核心票务测试通过。
- 锁牌互斥和释放占用测试通过。
- 游客公开页不能看到非公开字段。
- 结算、作废、拉黑、截图删除都有日志。
- 没有提交真实账号、密钥、token。
- `CHANGELOG.md` 已记录本次变更和验证命令。
